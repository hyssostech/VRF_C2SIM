## 2. The oracle contradiction (outranks section 1)

Two channels disagree about the one moving unit's direction. RPT (VR-Forces' own reports)
shows it east toward the objective; POS (WatchVrf dead-reckoned) shows it 63 m west then a
stationary sawtooth. On the two frozen units the channels agree exactly - so this is a
tracking failure specific to moving objects, not generic noise. Which channel is truthful is
NOT PROVEN offline; one POS sample at t=159.9 (run 161438Z) briefly matches RPT to 4.6 m,
favouring RPT, but n=1. Because POS displacement is the standing movement oracle, every past
negative movement result depends on resolving this.

The decisive test is a native change: log lastSetLocation() (raw) beside location()
(dead-reckoned) on the same row. It is NOT built - an attempt was reverted (5d14eda) for
breaking object creation. See section 5.
